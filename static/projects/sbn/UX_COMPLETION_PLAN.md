# SBN Completion Plan (Mostamal Hawaa)

## أهداف الخطة
- إنهاء رحلة المستخدم كاملة من التسجيل حتى نشر بوست مع مرفق (إعلان/منتج/خدمة/معرفة) ومتابعة تفاعل الآخرين.
- ضمان أن كل قسم (الإعلانات، المنتجات، الخدمات، المعرفة) يدعم الإضافة والاستعراض والتصفية بسهولة من الواجهة نفسها.
- رفع وضوح التصميم والتسلسل البصري بحيث يكون الانتقال بين الـ timeline والأقسام المتخصصة بديهيًا.

## حالة التنفيذ الحالية (مراجعة سريعة)
- يوجد كومبوزر موحد يدعم اختيار نوع المرفق (إعلان، منتج، خدمة، معرفة) مع حقول متخصصة لكل نوع، ويمكن نشر بوست أو ريل مع ملفات مرفوعة.
- هناك عرض للـ timeline يتضمن بطاقات بوستات مع مرفقات، بالإضافة إلى مقاطع Reels وأقسام مكدسة للمنتجات والخدمات والمعرفة، لكن بدون فصل سياقي واضح.
- ملفات البروفايل تعرض البيانات الأساسية (اسم، صورة، أرقام المتابعين) وبطاقات المحتوى، لكن لا توجد رحلة تحرير كاملة أو CTA واضحة للمتابعة/التواصل.

## فجوات حرجة يجب إغلاقها
1. **البروفايل وتجربة الهوية**
   - لا يوجد مسار واضح لتحديث الصورة الشخصية، الغلاف، والبيو، أو مراجعة قائمة الإعلانات/المنتجات/الخدمات التي نشرها المستخدم.
   - ينقصنا CTA ثابت (متابعة، رسالة، اتصال) وشارة الثقة/التحقق لتقليل الشكوك في المعاملات.

2. **إنشاء المحتوى والمرفقات**
   - اختيار المرفق يحتاج معاينة فورية (صورة مصغرة، عنوان، سعر/هاتف) مع منع النشر دون تأكيد الهدف المحدد.
   - دعم جاليري متعدد للمنتجات والإعلانات مع ترتيب الصور والعناوين الفرعية قبل النشر.
   - لا يوجد مسار حفظ مسودة أو استئناف لاحق، ما يربك المستخدم في الرحلات الطويلة (خدمة + صور + تفاصيل).

3. **عرض التفاصيل والاكتشاف**
   - تنقصنا شاشات/مودالات تفصيلية لكل نوع (إعلان مستعمل، منتج، خدمة، مقال معرفة) مع جاليري، CTA شراء/تواصل، ومعلومات البائع.
   - التبويب الحالي للصفحة الرئيسية غير مفصول؛ نحتاج فواصل واضحة بين Timeline اجتماعي، إعلانات مستعملة، منتجات/خدمات، ومعرفة، مع فلاتر فئات وهاشتاغ.

4. **الرحلة من التسجيل للنشر**
   - يجب توفير Onboarding مصغر (بطاقة مهام) يطلب: رفع صورة، اختيار فئات مفضلة، نشر أول بوست، وشرح سريع لأنواع المرفقات.
   - التدفق بعد التسجيل يجب أن يعيد المستخدم مباشرة إلى الكومبوزر مع حفظ المدخلات السابقة إن وجدت.

5. **الموثوقية والمتابعة**
   - إضافة عناصر الأمان: الإبلاغ/الحظر على البطاقات، شارة بائع موثق، وتوضيح سياسات الدفع/التسليم.
   - تحسين إشعارات التفاعل (تعليق/رسالة/حفظ) وربطها بصفحة إشعارات أو صندوق رسائل واضح.

## خطة التنفيذ المقترحة (4 أسابيع)
### الأسبوع 1: تثبيت الأساسيات
- تحسين الكومبوزر: معاينة المرفق + منع النشر بدون هدف + حفظ مسودة محلية.
- تحسين بروفايل: زر تعديل سريع، إمكانية تحديث الصورة/البيو، وإظهار تبويب المحتوى المنشور حسب النوع.
- إعداد تبويب علوي للفصل بين Timeline، إعلانات، منتجات/خدمات، معرفة.

### الأسبوع 2: شاشات التفاصيل والجاليري
- بناء مودال/صفحة تفاصيل للإعلان المستعمل (صور، سعر، موقع، هاتف، CTA اتصال/رسالة).
- بناء مودال/صفحة تفاصيل للمنتج/الخدمة مع جاليري وسعر/باقة/تقييمات.
- بناء قارئ مقالات معرفة بتنسيق مريح، صور غلاف، وروابط مقالات ذات صلة.

### الأسبوع 3: الرحلة الكاملة والتوعية
- إضافة بطاقة Onboarding في الهيرو مع مهام (صورة، فئات، أول بوست) وتلميحات حول المرفقات.
- دعم استئناف النشر بعد التسجيل (حفظ السياق)، وتوضيح نوع المرفق المختار في العنوان الفرعي للكومبوزر.
- إضافة فلاتر فئات/هاشتاغ تعمل بالفعل لكل تبويب.

### الأسبوع 4: الثقة والموثوقية
- دمج أزرار الإبلاغ/الحظر في بطاقات المحتوى والتفاصيل.
- إظهار شارة موثق للبائعين الموثوقين وتوضيح شروط الضمان/التسليم.
- تحسين إشعارات الردود/الحفظ وربطها بصفحة إشعارات أو Inbox بسيط.

## جاهزية الإطلاق (Checklist)
- الكومبوزر يمنع النشر بدون مرفق/هدف واضح ويعرض معاينة لكل نوع.
- كل نوع محتوى يملك صفحة/مودال تفاصيل مع جاليري وCTA تواصل/شراء.
- البروفايل يسمح بتعديل الصورة والبيو ويعرض محتوى المستخدم مجزأ حسب النوع.
- التبويبات والفلاتر تعمل وتعيد النتائج بالسياق الصحيح (Timeline، إعلان، منتج/خدمة، معرفة).
- تدفق التسجيل يعيد المستخدم إلى الكومبوزر ويحفظ إدخالاته السابقة.
- إشعارات وتبليغ/حظر تعمل من البطاقات والتفاصيل.

## ما تم تنفيذه الآن
- تم تفعيل رحلة إرشاد على الصفحة الرئيسية (Onboarding) بمهام عملية: رفع الصورة، إكمال البايو، نشر أول بوست، وتجربة مرفق.
- تم إضافة محرر ملف شخصي مباشر (صورة/بايو/اسم) مع حفظ التقدم في المهام وإغلاقها من نفس الواجهة.
- تمت إضافة حفظ مسودة الكومبوزر محليًا بحيث تستأنف الكتابة والمرفقات بعد إعادة الدخول أو التنقل بين الأقسام.
- أُضيفت شاشات تفصيلية للمرفقات مع جاليري، بيانات بائع، أزرار مراسلة/اتصال، وبطاقات أمان واضحة لكل إعلان/منتج/مقال.
- فُعلت طبقة الثقة (شارة موثوق) وأزرار الإبلاغ/الحظر من الكروت والأوفرلاي، مع نموذج بلاغ سريع يحفظ السياق.
- أضيفت طبقة مراسلة سريعة (Contact overlay) مع حفظ رقم الهاتف، بالإضافة إلى أزرار متابعة ورسائل من ملف المستخدم.
- تمت إضافة فلاتر فعّالة (هاشتاغ/تصنيف/حالة/بحث) مع ملخص قابل للمسح، كما تم فصل محتوى البروفايل إلى تبويبات (بوستات، إعلانات، منتج/خدمة، معرفة) مع جاليريات واضحة وعرض تفاصيل سريع.
